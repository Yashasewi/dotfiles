03 SSH Config

~/.ssh/config 
# Work GitHub (default)
Host github.com
	HostName github.com
	User git
	IdentityAgent "~/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock"
	IdentityFile ~/.ssh/work_github.pub
	IdentitiesOnly yes

# Personal GitHub
Host github.com-personal
	HostName github.com
	User git
	IdentityAgent "~/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock"
	IdentityFile ~/.ssh/personal_github.pub
	IdentitiesOnly yes

# Everything else still uses 1Password agent
Host *
	IdentityAgent "~/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock"
ℹ️

IdentitiesOnly yes is critical. Without it, SSH offers ALL keys from the agent and GitHub accepts whichever matches first (non-deterministic). With it, SSH only offers the key specified in IdentityFile.

04 1Password Agent Config

~/.config/1Password/ssh/agent.toml 
# Work vault keys
[[ssh-keys]]
item = "GitHub"
vault = "Work"

[[ssh-keys]]
item = "GitHub Signing Key"
vault = "Work"

# Personal vault keys
[[ssh-keys]]
item = "GitHub"
vault = "Personal"

[[ssh-keys]]
item = "GitHub Signing Key"
vault = "Personal"
⚠️

This file MUST end with a trailing newline. Without it, 1Password silently reverts to default behavior (Private vault only).

Key points:

  • Order controls which keys are offered first
  • Using item pins to specific keys (vs vault alone which exposes all keys in that vault)
  • Changes take effect immediately (no restart needed)

05 Git Config (Global)

~/.gitconfig 
[gpg]
	format = ssh
[gpg "ssh"]
	program = /Applications/1Password.app/Contents/MacOS/op-ssh-sign
[commit]
	gpgsign = true
[user]
	signingkey = ssh-ed25519 AAAA...your-work-signing-key...
	name = Your-Work-Username
	email = you@company.com
[includeIf "gitdir:~/Desktop/Personal/"]
	path = ~/.gitconfig-personal
[pull]
	rebase = false
[init]
	defaultBranch = main
[credential "https://github.com"]
	helper = 
	helper = !/opt/homebrew/bin/gh auth git-credential
[credential "https://gist.github.com"]
	helper = 
	helper = !/opt/homebrew/bin/gh auth git-credential
ℹ️

The includeIf "gitdir:..." directive loads the personal config ONLY when working inside ~/Desktop/Personal/. The path is case-sensitive.

06 Git Config (Personal)

~/.gitconfig-personal 
[user]
	name = Your-Personal-Username
	email = 12345678+username@users.noreply.github.com
	signingkey = ssh-ed25519 AAAA...your-personal-signing-key...
[url "git@github.com-personal:"]
	insteadOf = git@github.com:

The url.insteadOf rule is the glue. When you clone git@github.com:your-personal-username/repo.git from inside ~/Desktop/Personal/, git silently rewrites the remote to git@github.com-personal:your-personal-username/repo.git, which routes through the personal SSH host alias.

07 Public Key Files

These files tell SSH which specific key to request from the 1Password agent. Without them, the agent offers all keys non-deterministically.

~/.ssh/work_github.pub 
ssh-ed25519 AAAA...your-work-auth-public-key... GitHub
~/.ssh/personal_github.pub 
ssh-ed25519 AAAA...your-personal-auth-public-key... GitHub
⚠️

Permissions MUST be chmod 644. If set to 600, SSH treats them as private keys and throws "invalid format" or "UNPROTECTED PRIVATE KEY FILE" errors.

To regenerate these files from the agent:

SSH_AUTH_SOCK=~/Library/Group\ Containers/2BUA8C4S2C.com.1password/t/agent.sock ssh-add -L

Copy the relevant line into the appropriate .pub file.

08 How It Works (Step by Step)

You git push from ~/Desktop/Personal/some-repo/

Git sees gitdir matches ~/Desktop/Personal/ and loads .gitconfig-personal

The url.insteadOf rule rewrites git@github.com: to git@github.com-personal:

SSH matches Host github.com-personal in ~/.ssh/config

IdentitiesOnly yes + IdentityFile ~/.ssh/personal_github.pub tells SSH to ONLY request the personal key from 1Password

1Password prompts Touch ID, then signs with the matching private key from the Personal vault

GitHub authenticates the connection as your-personal-username

For commit signing: op-ssh-sign uses the personal signingkey from .gitconfig-personal via 1Password

09 Common Commands

Test SSH Authentication

# Should say: Hi your-work-username!
ssh -T git@github.com

# Should say: Hi your-personal-username!
ssh -T git@github.com-personal

List Agent Keys

SSH_AUTH_SOCK=~/Library/Group\ Containers/2BUA8C4S2C.com.1password/t/agent.sock ssh-add -l

Verify Identity in a Repo

# Check which identity a repo will use
git -C ~/Desktop/Personal/some-repo config user.email
git -C ~/Desktop/Personal/some-repo config user.name
git -C ~/Desktop/Personal/some-repo config user.signingkey

Clone a Personal Repo

# From inside ~/Desktop/Personal/ — url.insteadOf handles rewriting
cd ~/Desktop/Personal
git clone git@github.com:your-personal-username/repo-name.git

GitHub CLI (Multiple Accounts)

# Add personal account
gh auth login

# Switch between accounts
gh auth switch

# Check current status
gh auth status

Debug SSH Connection

# Verbose output shows which key is being offered
ssh -vT git@github.com-personal 2>&1 | grep "Will attempt\|Offering\|Authenticated"

10 Troubleshooting

ProblemCauseFix
Permission denied (publickey) 1Password agent not serving the key Check agent.toml has the correct vault/item. Run ssh-add -l to verify key is listed.
Wrong account authenticates IdentitiesOnly yes missing, or wrong key in .pub file Verify .pub file matches the correct key from ssh-add -L output. Ensure IdentitiesOnly yes is set.
invalid format or UNPROTECTED PRIVATE KEY FILE .pub file has wrong permissions chmod 644 ~/.ssh/*.pub
Commits not showing as "Verified" on GitHub Signing key not added to GitHub GitHub Settings → SSH and GPG keys → New SSH key → Type: "Signing Key" → paste the public signing key
Personal config not loading Case mismatch in gitdir path The path is case-sensitive. Must match exactly: ~/Desktop/Personal/ (capital P). Trailing slash required.
No Touch ID prompt from 1Password 1Password app not running or locked Open 1Password and unlock it. The SSH agent requires the app to be running.
agent.toml changes not taking effect Missing trailing newline Ensure file ends with a blank line. 1Password silently ignores malformed config.
